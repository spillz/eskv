# ESKV User Guide

This guide outlines the documentation set for building applications with ESKV. Each section links to a dedicated document that dives deeper into the topic. Start with the overview if you are new to the framework, then explore the modules that match your project needs.

1. [Framework Overview](overview.md) – Understand the goals of ESKV, how the canvas-based renderer works, and the relationship between apps, widgets, and resources.
2. [Project Setup](getting-started.md) – Install dependencies, serve the samples, and wire the library into a bundler or static page.
3. [Core Concepts](core-concepts.md) – Learn about the event loop, widget tree lifecycle, resource management, and animation model.
4. [Widget Catalog](widgets.md) – Explore built-in widgets such as labels, buttons, sliders, notebooks, and modal views.
5. [Layout System](layouts.md) – Arrange widgets using box layouts, grids, scroll views, and notebooks for complex UIs.
6. [Input Handling](input.md) – Route pointer, touch, and keyboard events using the unified input module.
7. [Text and Rich Content](text-and-rich-content.md) – Format text with markup, measure fonts, and render rich text blocks.
8. [Styling and Colors](styling-and-colors.md) – Work with color utilities and theming strategies for consistent visuals.
9. [Graphics, Sprites, and Tile Maps](graphics-and-sprites.md) – Draw animated scenes with sprite batches, tile maps, and WebGL helpers.
10. [Animation, Timing, and Effects](timers-and-animation.md) – Schedule work with timers, drive widget animations, and coordinate transitions.
11. [Utility Modules](utilities.md) – Reuse geometry, math, randomness, and helper classes across your application logic.
12. [Integration and Tooling](integration-and-tooling.md) – Combine ESKV with build tools, asset pipelines, and testing strategies.

> 💡 Documentation stubs are intentionally brief today. Flesh them out as you explore each module or add new capabilities to the framework.
